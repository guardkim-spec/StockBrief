StockBrief PRD v2.0
1. 프로젝트 개요
항목내용이름StockBrief한 줄 설명매일 오후 9시 30분, 한국·미국 주식 뉴스와 시세를 자동 분석해 섹터 인사이트를 제공하는 일일 주식 브리핑 서비스유형대시보드 + 이메일 뉴스레터난이도중-상버전v2.0

2. 사용자 시나리오
항목내용누가개인 사용자 (본인 단독)언제, 왜매일 오후 9시 30분, 당일 한국·미국 주식 시장의 흐름을 빠르게 파악하고 다음날 투자 판단에 참고하기 위해 사용

3. MVP 단계 정의
1차 (핵심 파이프라인)

한국/미국 뉴스 수집 및 섹터 분류
한국/미국 거래대금 상위 100위 수집 및 섹터 분류
자동 이메일 리포트 발송

2차 (대시보드)

React 대시보드 구축
캔들차트 + 추세 분석
AI 섹터 추천
한미 연계 분석

3차 (고도화)

백테스팅
정확도 기반 모델 개선
이메일 리포트 미리보기 화면

16. 제약사항
기술적 제약

Gemini 1.5 Flash 무료 티어 일일 100만 토큰 한도 내 운영
NewsAPI 무료 플랜 일 100회 요청 한도 준수
yfinance / 네이버 뉴스 RSS 잦은 요청 차단 방지를 위해 랜덤 딜레이 필수 적용
pykrx 데이터는 장 마감 후 (오후 3:30 이후) 확정 데이터만 수집 가능
미국 주식 데이터는 KST 오후 7시 이후 당일 데이터 수집 가능 (pytz로 서머타임 자동 처리)

데이터 제약

뉴스 분석은 당일 기사만 대상 (과거 소급 분석 없음)
거래대금 상위 100위는 당일 장 마감 기준 데이터
20일 일봉 데이터는 영업일 기준 (공휴일 제외)
한국/미국 공휴일에는 데이터 수집 자동 스킵 후 휴장일 안내 이메일 발송
이상값 (등락률 ±30% 초과) 데이터는 플래그 처리 후 별도 표시

운영 제약

개인 사용자 단독 운영으로 별도 서버 비용 최소화
GitHub Actions 무료 티어 월 2,000분 한도 내 운영 (예상 월 사용량 약 900분)
Google Sheets 데이터 보관 기간: 최대 1년 (용량 관리)
이메일 발송 실패 시 재시도 1회, 이후 수동 확인

법적/윤리적 제약

수집 데이터는 개인 투자 참고용으로만 사용
뉴스 원문 재배포 금지, 링크 제공 방식으로만 제공
투자 추천이 아닌 정보 제공 서비스임을 명시
